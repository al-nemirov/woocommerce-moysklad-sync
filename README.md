# WooCommerce МойСклад Sync

Синхронизация заказов WooCommerce с [МойСклад](https://www.moysklad.ru/) через JSON API 1.2.

## Возможности

- **Экспорт заказов** WC → МС как заказы покупателя (черновики, `applicable: false`)
- **Автоотправка** при смене статуса WC (настраиваемый триггер)
- **Ручная отправка** из мета-бокса в карточке заказа
- **Контрагенты** — поиск по email/телефону, автосоздание нового
- **Товары** — сопоставление по артикулу (SKU), товар-заглушка для непривязанных
- **Вебхук МС → WC** — двусторонняя синхронизация статусов
- **Маппинг статусов** — настраиваемое соответствие статусов МС ↔ WC
- **Колонка «МС»** в списке заказов (✓ / ✗ / —)
- **AJAX проверка подключения** и загрузка организаций/складов/статусов из API
- **Совместимость** с HPOS (WooCommerce Custom Order Tables)
- **Debug-режим** — логирование API-запросов в `debug.log`
- **Миграция** настроек со старого плагина (yd_moysklad_*)

## Установка

1. Скачайте ZIP из [Releases](https://github.com/al-nemirov/woocommerce-moysklad-sync/releases)
2. WordPress → Плагины → Добавить → Загрузить плагин
3. Активируйте
4. **WooCommerce → МойСклад** → введите логин/пароль → «Проверить подключение»

## Настройки

**WooCommerce → МойСклад:**

### Подключение

| Параметр | Описание |
|---|---|
| Логин (email) | Email аккаунта МойСклад |
| Пароль | Пароль аккаунта |
| Проверка | AJAX-кнопка для проверки подключения к API |

### Организация и склад

| Параметр | Описание |
|---|---|
| Организация | Выбор из списка (загружается кнопкой «Загрузить из МойСклад») |
| Склад | Привязка заказов к конкретному складу |

### Автоотправка

| Параметр | Описание |
|---|---|
| Отправлять при статусе | Триггер автоотправки (по умолчанию — «Обработка») |
| Начальный статус МС | В каком статусе создаётся заказ в МойСклад |

### Товары

| Параметр | Описание |
|---|---|
| Товар-заглушка (UUID) | UUID товара в МС для позиций без совпадения по SKU |

### Маппинг статусов

Настройте соответствие статусов МС → WC. При изменении статуса в МойСклад — статус WooCommerce обновится автоматически через вебхук.

### Дополнительно

| Параметр | Описание |
|---|---|
| Метка в примечаниях | Префикс в примечаниях к заказу (по умолчанию «МойСклад») |
| Debug | Логирование API-запросов/ответов в `wp-content/debug.log` |

## Вебхук (МС → WC)

Для двусторонней синхронизации статусов:

1. Настройте маппинг статусов МС → WC на странице настроек
2. Нажмите «Создать вебхук в МойСклад»
3. При изменении статуса заказа в МС — статус WC обновится автоматически

URL вебхука: `https://ваш-сайт/wp-json/moysklad/v1/webhook`

## Мета-бокс в заказе

В карточке заказа WooCommerce появляется панель «МойСклад»:
- **Не отправлен** — кнопка «Отправить в МойСклад»
- **Передан** — ссылка на заказ в МС + кнопка «Пересоздать»
- **Ошибка** — текст ошибки + кнопка повторной отправки

## Колонка в списке заказов

| Символ | Значение |
|---|---|
| ✓ (зелёный) | Заказ передан в МС |
| ✗ (красный) | Ошибка синхронизации |
| — (серый) | Не отправлялся |

## Требования

- WordPress 5.8+
- WooCommerce 5.0+
- PHP 7.4+
- Аккаунт МойСклад с доступом к API
